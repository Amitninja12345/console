数据源子系统允许您创建和配置数据源和管理 JDBC 数据库驱动程序。

数据源

两种常见的资源类型被称为<emphasis>数据源</emphasis>和<emphasis>XA 数据源</emphasis>。

  • Non-XA datasources 用于不使用事务的应用程序，或使用带有一个数据库的事务的应用程序。
  • XA 数据源用于其交易涉及多个数据库的分布式应用程序。XA数据源会产生额外的开销。

JDBC 驱动程序

在您的应用程序可以连接到数据源之前，需要安装供应商的 JDBC 驱动程序。您可以选择以下两种方法安装 JDBC 驱动程序：

模块

要安装J DBC 驱动程序作为一个模块，需要在${build.installDir}/modules 中创建一个文件路径结构，把 JDBC 驱动 JAR 复制到 main/ 子目录，并创建一个 module.xml 文件。

如果 JDBC 驱动程序作为一个模块被提供，您可以使用此部分添加和删除驱动程序配置。

部署

您可以象其它部署一样来部署 JDBC 驱动程序。这意味着，如果您使用一个管理域，则可以在服务器组中跨多个服务器部署它们。任何JDBC 4标准的驱动程序都会按照它们的名称和版本自动识别并安装到系统中。

在域模式中，作为应用程序部署的驱动程序将只在有运行的服务器与所选配置集相匹配的情况下，才在本节中显示。
