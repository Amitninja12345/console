JBoss Remoting possède trois éléments configurables de niveau supérieur : le pool de threads de travail, un ou plusieurs connecteurs et une série d'URI de connexion locale et distante.

La plupart d’entre vous n'auront pas besoin de configurer le sous-système Remoting, à moins qu'ils n'utilisent des connecteurs personnalisés pour leurs propres applications. Les applications qui agissent comme clients Remoting, comme les EJBs, ont besoin d'une configuration séparée pour se connecter à un connecteur spécifique.
