Extensions are a way to add new features to the management console. They are written in JavaScript and can use the JavaScript API to interact with the console and the management interface. If you want to develop an extension, see https://github.com/hal/hal.next/wiki/Extensions for more information.

Extensions are written in JavaScript and are injected into the browser. Please make sure you install only extensions that you trust!

Extension Points

The console provides three different extension points which can be used by extensions:

  1. Header: Adds a menu item to the "Extensions" dropdown in the header
  2. Finder Item: Adds a new item to a specific finder column
  3. Footer: Adds a menu item to the "Extensions" dropdown in the footer

Deployment

Extensions consists of one JavaScript file and a list of optional stylesheets. They come in two different flavours:

Bundled Extensions

Bundled extensions are part of the WildFly installation and installed as WildFly modules. The module contains the extension's script and stylesheets. Bundled extensions are read-only in the console.

Standalone Extensions

Standalone extensions are hosted by a public available endpoint which serves the extension's script and stylesheets. You can add and remove standalone extensions using the management console. They're stored in the browser's local storage. As such they're scoped to the browser and URL which runs the management console.
