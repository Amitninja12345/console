Un conteneur de servlet fournit toutes les configurations de servlet, de JSP et de socket web, y compris la configuration de session. Alors que la plupart des serveurs n'auront besoin que d'un seul conteneur de servlet, il est possible de configurer plusieurs conteneurs de servlet en ajoutant un élément servlet-conteneur supplémentaire. Le fait d'avoir plusieurs conteneurs de servlets permet un comportement tel que le déploiement de plusieurs déploiements sur le même chemin de contexte sur différents hôtes virtuels.

Remarque
Une grande partie de la configuration fournie par le conteneur de servlet peut être écrasée individuellement par les applications déployées en utilisant leur fichier web.xml.
