La méthode d'application d'un correctif à ${build.shortName} dépend de votre méthode d'installation. Si vous avez installé ${build.shortName} en utilisant la méthode ZIP ou la méthode d'installation, vous devez utiliser le système de gestion des correctifs basé ZIP. Si vous avez utilisé RPMs pour installer ${build.shortName} sur Red Hat Enterprise Linux, vous devez utiliser des correctifs RPM. Important.

Avant d'appliquer ou d'annuler un correctif, vous devez sauvegarder votre serveur ${build.shortName}, y compris tous les déploiements et les fichiers de configuration.

Des correctifs cumulatifs pour une installation ZIP ou Installer de ${build.shortName} sont disponibles pour téléchargement à partir du Portail Client Red Hat.

Pour plusieurs hôtes ${build.shortName} dans un environnement de domaine géré, les hôtes individuels peuvent être patchés à partir de votre contrôleur de domaine ${build.shortName}.

En plus d'appliquer un patch, vous pouvez également faire reculer l'application d'un correctif.

Remarques importantes sur le correctif d'installation de ZIP/Installer

  • Si vous appliquez un correctif qui met à jour un module, les nouveaux JAR patchés qui sont utilisés lors de l'exécution seront stockés dans ${build.installDir}/modules/system/system/layers/base/.overlays/PATCH_ID/MODULE. Les fichiers originaux non corrigés sont laissés dans ${build.installDir}/modules/system/system/layers/base/MODULE, mais ces JAR ne sont pas utilisés au moment de l'exécution.
  • Afin de diminuer significativement la taille des correctifs cumulatifs pour ${build.shortName}, vous ne pouvez plus effectuer un retour partiel d'un correctif cumulatif. Pour un correctif déjà appliqué, vous ne pourrez pas revenir en arrière.
  • Par exemple, si vous appliquez CP03 à ${build.shortName}, vous ne pourrez pas revenir à CP01 ou CP02. Si vous souhaitez pouvoir revenir sur chaque correctif cumulatif, chaque patch cumulatif devra être appliqué séparément dans l'ordre dans lequel ils sont sortis.
