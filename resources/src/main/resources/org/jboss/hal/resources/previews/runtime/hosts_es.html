A host controller is launched when the domain.sh or domain.bat script is run on a host.

La responsabilidad principal de un controlador host es la administración de servidores. Delega tareas de administración de dominio y es el responsable de iniciar y detener los procesos del servidor de aplicaciones individuales que ejecutan en su host.

Interactúa con el controlador de dominios para ayudar a administrar la comunicación entre los servidores y el controlador de dominios. Múltiples controladores host de un dominio pueden interactuar con un solo controlador de dominios. Por lo tanto, todos los controladores host e instancias de servidor ejecutando en un solo modo de dominios tienen un solo controlador de dominios y deben pertenecer al mismo dominio.

By default each host controller reads its configuration from the domain/configuration/host.xml file located in the unzipped ${build.shortName} installation file on its host's filesystem. The host.xml file contains the following configuration information that is specific to the particular host:

  • The names of the ${build.shortName} instances meant to run from this installation.
  • Cualquiera de las siguientes configuraciones:
    • La manera en que el controlador host contacta el controlador de dominios para registrarse a si mismo y acceder a la configuración del dominio.
    • La manera de encontrar y contactar un controlador de dominios remoto.
    • That the host controller is to act as the domain controller
  • Configurations specific to the local physical installation. For example, named interface definitions declared in domain.xml can be mapped to an actual machine-specific IP address in host.xml. And abstract path names in domain.xml can be mapped to actual filesystem paths in host.xml.
